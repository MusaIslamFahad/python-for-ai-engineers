## 🌟 Why This Repository?

Most Python tutorials teach you Python. This one teaches you **Python the way AI Engineers actually use it.**

Every concept is introduced with a real-world AI/ML context. Every example is designed to prepare you for the kinds of problems you'll face when building LLM pipelines, training neural networks, designing data systems, and shipping AI products.

Whether you're a complete beginner or a developer transitioning into AI, this is the curriculum you wish existed when you started.

### 01 — Basics

> Build the foundation that every AI Engineer stands on.

- Variables, Data Types & Type Conversion
- Arithmetic, Comparison & Logical Operators
- `if`/`elif`/`else` - Control Flow
- `for` and `while` Loops - Iteration Patterns
- Functions - Arguments, `*args`, `**kwargs`, Return Values
- String Manipulation - f-strings, `.split()`, `.join()`, slicing
- Lists & List Comprehensions - the workhorse of data pipelines
- Dictionaries & Sets - fast lookups, key-value patterns
- Tuples & Immutability
- File I/O - reading CSVs, writing logs, handling paths
- Error Handling - `try/except/finally`, custom exceptions
- Modules & Packages - `import`, `pip`, structuring your code

---

### 02 — Intermediate

> Level up to write production-quality, reusable Python code.

- Object-Oriented Programming - Classes, Inheritance, Polymorphism, Dunder Methods
- Decorators - `@staticmethod`, `@classmethod`, custom decorators, `functools.wraps`
- Generators & Iterators - `yield`, lazy evaluation, memory-efficient pipelines
- Context Managers - `with` statement, `__enter__` / `__exit__`, `contextlib`
- Regular Expressions - `re` module, pattern matching, parsing log files & text data
- Lambda Functions & Functional Programming - `map`, `filter`, `reduce`
- Working with JSON, CSV & APIs
- Python's Standard Library Deep Dive

---

### 03 — Advanced

> Write high-performance, maintainable Python like a senior engineer.

- Type Hints & `mypy` - annotating functions, generics, `TypeVar`
- Async Python - `asyncio`, `await`, `aiohttp`, concurrent API calls
- Multiprocessing & Threading - CPU-bound vs. I/O-bound tasks
- Design Patterns for AI Systems - Factory, Singleton, Observer, Strategy
- Testing - `pytest`, fixtures, mocking, test-driven AI workflows
- Python Performance - profiling, `cProfile`, `line_profiler`, `numba`
- Memory Management - garbage collection, `__slots__`, weak references
- Packaging & Project Structure - `pyproject.toml`, `setup.cfg`, `__init__.py`

---

### 04 — Data Science Stack

> Master the libraries that power every ML project.

- **NumPy** - arrays, broadcasting, vectorized operations, linear algebra
- **Pandas** - DataFrames, `groupby`, `merge`, `pivot_table`, handling missing data
- **Matplotlib** - figures, axes, subplots, publication-quality plots
- **Seaborn** - statistical visualizations, heatmaps, pair plots
- Data Cleaning Workflows - outlier detection, normalization, encoding
- Exploratory Data Analysis (EDA) - patterns, correlations, distributions

---

### 05 — ML Fundamentals

> Understand machine learning from a practitioner's perspective.

- Scikit-learn Ecosystem - estimators, transformers, pipelines
- Supervised Learning - Linear Regression, Logistic Regression, Decision Trees, SVMs
- Unsupervised Learning - K-Means, DBSCAN, PCA, t-SNE
- Preprocessing - `StandardScaler`, `LabelEncoder`, `OneHotEncoder`, imputation
- Model Evaluation - cross-validation, confusion matrix, ROC/AUC, precision/recall
- Hyperparameter Tuning - `GridSearchCV`, `RandomizedSearchCV`, Optuna
- Feature Engineering & Selection
- Scikit-learn Pipelines - end-to-end reproducible ML

---

### 06 — Deep Learning

> Build and train neural networks from scratch to state-of-the-art.

- **PyTorch** - tensors, autograd, custom datasets, DataLoaders
- Building Neural Networks - `nn.Module`, layers, activations, loss functions
- Training Loops - forward pass, backward pass, optimizers (`Adam`, `AdamW`)
- CNNs - image classification, transfer learning with ResNet/EfficientNet
- RNNs & LSTMs - sequence modeling, time-series prediction
- Transformers - attention mechanism, positional encoding, multi-head attention
- **Keras/TensorFlow** - high-level API, `Sequential`, `Functional`, `Model.fit()`
- Model Saving, Loading & Deployment
- GPU Acceleration - CUDA, `device` management, mixed precision training

---

### 07 — AI Engineering

> Build production AI systems with LLMs, RAG, and Agents.

- LLM APIs - OpenAI, Anthropic Claude, Google Gemini integration
- Prompt Engineering - zero-shot, few-shot, chain-of-thought, structured output
- Embeddings - `text-embedding-3-small`, Voyage, Cohere; cosine similarity
- Vector Databases - Chroma, Pinecone, Weaviate - indexing, querying, filtering
- RAG (Retrieval-Augmented Generation) - chunking, retrieval, re-ranking, evaluation
- LangChain - chains, memory, tools, document loaders
- LlamaIndex - index construction, query engines, data connectors
- Agents - ReAct, tool use, multi-agent patterns, AutoGen, CrewAI
- Structured Output - JSON mode, function calling, Pydantic validation
- Streaming Responses - SSE, WebSockets with LLMs

## 📓 Notebook Structure

Every single notebook in this repo follows the same battle-tested template — so you always know where to look:

```
📌 What is it?                - Plain-English definition (no jargon first)
🧠 Why AI Engineers need it   - Concrete real-world AI/ML context
✏️  Syntax                    - Minimal, clean syntax reference
💡 Examples                   - 3–5 progressive examples (simple → real-world)
⚠️  Common Mistakes           - The gotchas that trip up even experienced devs
🔗 What's Next?               - Link to the next concept in the path
```

## 🚀 Quick Start

### Option 1: Google Colab (No Setup Required) ✅

Click the **Open in Colab** badge inside any notebook to run it instantly in your browser.

### Option 2: Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/MusaIslamFahad/python-for-ai-engineers.git
cd python-for-ai-engineers

# 2. (Optional but recommended) Create a virtual environment
python -m venv venv
source venv/bin/activate       # On Windows: venv\Scripts\activate

# 3. Install all dependencies
pip install -r requirements.txt

# 4. Launch Jupyter
jupyter notebook
```

### Option 3: GitHub Codespaces

Click **Code → Codespaces → Create codespace on main** and get a cloud VS Code environment with everything pre-installed.

## 🤝 Contributing

Contributions make this repo better for everyone. Here's how to get involved:

1. **Fork** this repository
2. **Create** a branch: `git checkout -b feature/your-notebook-topic`
3. **Add** your notebook following the [notebook template](#-notebook-structure)
4. **Commit** your changes: `git commit -m "Add: notebook on topic X"`
5. **Push** to your fork: `git push origin feature/your-notebook-topic`
6. **Open a Pull Request** with a short description

Please read [CONTRIBUTING.md](./CONTRIBUTING.md) for detailed guidelines on notebook quality, formatting standards, and the review process.

### Ideas for Contributions
- New notebooks for any existing module
- Real-world project ideas for Module 08
- Bug fixes or typo corrections
- Translations of notebooks into other languages
- Better examples or edge case coverage
